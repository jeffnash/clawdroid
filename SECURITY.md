# Security Policy

## Reporting a vulnerability

Please do not open public issues for security-sensitive bugs.

Use GitHub Security Advisories if the repository has them enabled. If not, contact the project maintainer through a private channel and include:

- affected component
- host/guest environment details
- reproduction steps
- impact
- any logs or screenshots needed to reproduce safely

## Scope

Security-relevant areas in this project include:

- the local HTTP daemon
- ADB/Waydroid control surfaces
- plugin tool exposure and approval gates
- APK download/install flows
- any host-control or privileged setup scripts

Reports about packaging drift, broken automation, or test failures should go through normal issues instead.
